## @onruntime/translations क्या है?

`@onruntime/translations` एक हल्का, प्रकार-सुरक्षित अंतर्राष्ट्रीयकरण (i18n) पुस्तकालय है जो React और Next.js अनुप्रयोगों के लिए डिज़ाइन किया गया है। यह पूर्ण TypeScript समर्थन के साथ अनुवाद प्रबंधन के लिए एक सरल API प्रदान करता है।

<Callout type="info" title="मुख्य विशेषताएँ">
  - TypeScript के साथ प्रकार-सुरक्षित अनुवाद
  - React और Next.js (ऐप राउटर) का समर्थन
  - Namespace-आधारित संगठन
  - वैरिएबल इंटरपोलेशन
  - हल्का और प्रदर्शन में उत्कृष्ट
</Callout>

## इसका उपयोग क्यों करें?

### प्रकार सुरक्षा

अमान्य अनुवाद कुंजी का उपयोग करने पर संकलन समय की त्रुटियाँ प्राप्त करें:

```typescript
const { t } = useTranslation("common");

t("greeting");        // ✅ वैध कुंजी
t("invalid_key");     // ❌ TypeScript त्रुटि
```

### सरल API

आपकी सभी अनुवाद आवश्यकताओं के लिए केवल दो हुक हैं:

```typescript
// क्लाइंट कंपोनेंट
const { t } = useTranslation("namespace");

// सर्वर कंपोनेंट (Next.js)
const { t } = await getTranslation("namespace");
```

### व्यवस्थित अनुवाद

अपने अनुवाद को नामस्थान के साथ संगठित रखें:

<Files>
  <Folder name="locales" defaultOpen>
    <Folder name="en" defaultOpen>
      <File name="common.json" />
      <File name="auth.json" />
      <File name="dashboard.json" />
    </Folder>
    <Folder name="fr">
      <File name="common.json" />
      <File name="auth.json" />
      <File name="dashboard.json" />
    </Folder>
  </Folder>
</Files>

## त्वरित उदाहरण

```tsx
// locales/en/common.json
{
  "greeting": "Hello, {name}!"
}

// कंपोनेंट
import { useTranslation } from "@onruntime/translations/react";

function Welcome({ name }) {
  const { t } = useTranslation("common");

  return <h1>{t("greeting", { name })}</h1>;
}
```
