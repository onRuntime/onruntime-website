## Structuur

De commit-bericht moet als volgt gestructureerd zijn en **in kleine letters**:

```
<gitmoji> <type> <beschrijving> [(#<issue nummer>)]

[optionele body]

[optionele footer(s)]
```

### Voorbeeld

```bash
ğŸ“ update documentatie bijdragers

- Voeg @jerembdn toe als bijdrager
- Voeg @younesbessa toe als bijdrager

Co-authored-by: Younes Bessa <younes@onruntime.com>
```

De meeste tijd is een eenvoudige one-liner genoeg:

```bash
ğŸ“ update documentatie bijdragers
```

## Types

We gebruiken de volgende commit types:

| Type | Beschrijving |
| --- | --- |
| `add` | Voeg een nieuwe functie toe |
| `fix` | Los een bug op |
| `improve` | Verbeter iets |
| `update` | Update iets |
| `remove` | Verwijder iets |
| `refactor` | Refactor iets |
| `rename` | Hernoem iets |
| `move` | Verplaats een bestand of map |
| `upgrade` | Upgrade afhankelijkheden |
| `downgrade` | Downgrade afhankelijkheden |

## Beschrijving

Schrijf beschrijvingen in **imperatieve vorm** en **kleine letters**:

```bash
# Goed
ğŸ“ update documentatie bijdragers

# Slecht
ğŸ“ updated documentatie bijdragers

# Slecht
ğŸ“ Update documentatie bijdragers
```

## Issue Nummer

Koppel commits aan issues door het issue nummer toe te voegen:

```bash
ğŸ“ update documentatie bijdragers (#123)
```
