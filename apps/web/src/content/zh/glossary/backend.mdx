## 定义

后端就像是店面后面的工厂！它是所有在 onRuntime 上运行我们应用程序的隐形部分。比如，当你使用 Tonight Pass 时，你与美观而直观的界面进行互动，但其实是我们的后端在完成所有繁重的工作：验证你的凭证、存储事件数据、计算推荐等等。

## 我们的日常工作

### 服务器
我们的后端工程师花时间优化我们的服务器。随着 Tonight Pass 的流量增加，我们花了很多时间优化一切！我们主要使用 AWS 和 Google Cloud 来托管我们的应用程序。

### API
这是我们的专长！我们创建强大的 API，让我们的前端能够准确地获取所需的信息。我们其实更喜欢 REST API，尽管在某些项目上，我们与 GraphQL 玩得越来越开心。

### 数据库
我们存储一切的地方！根据项目的不同，我们在 MongoDB（用于 Tonight Pass）、PostgreSQL（用于需要复杂关系的项目）和有时用 Redis 来进行缓存之间进行权衡。选择数据库至关重要，我们总是事先考虑这一点。

### 业务逻辑
我们工作的核心。将客户的需求转化为可工作的代码。例如，对于 Tonight Pass 的推荐算法，我们花了几周时间来完善规则和优化查询。

## 我们最喜欢的后端技术栈

### 编程语言
- **Node.js/TypeScript**：我们最喜欢的组合！几乎在所有最近的项目中都使用它
- **Python**：当我们需要进行数据分析或机器学习时非常合适
- **Go**：我们为某些需要性能的微服务采用了它

### 框架
- **NestJS**：我们最喜欢的大型应用的 Node.js 框架
- **Express**：用于更简单的 API 或原型
- **Django**：当我们需要快速构建一个强大的管理后台时

## 为什么这点很重要

后端就像房子的基础。如果基础做得不好，一切都会崩溃。在 onRuntime，我们特别自豪于：

- 我们的架构能够处理 Tonight Pass 的流量高峰（感谢跨年派对！）
- 我们的安全性完美无缺（敲木头）
- 我们超快的响应时间

对于客户来说，这往往是最不显眼的部分，但正是它给我们带来了最多的头疼...以及当一切运转如意时，最满意的感觉！
