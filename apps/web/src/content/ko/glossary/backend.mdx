## 정의

백엔드는 상점 뒤에 있는 공장 같은 거야! 우리의 애플리케이션을 onRuntime에서 구동하는 보이지 않는 모든 부분이야. 예를 들어 Tonight Pass를 사용할 때, 너는 아름답고 직관적인 인터페이스와 상호작용하지만, 모든 힘든 작업은 우리 백엔드가 하고 있어: 너의 자격증명을 확인하고, 이벤트 데이터를 저장하고, 추천을 계산하는 등의 작업이야.

## 우리가 매일 하는 일

### 서버
우리 백엔드 엔지니어들은 서버 최적화에 시간을 보내고 있어. Tonight Pass의 트래픽이 증가하면서 모든 것을 최적화하는 데 많은 시간을 쏟아야 했어! 우리는 주로 AWS와 Google Cloud를 사용해서 애플리케이션을 호스팅해.

### API
이건 우리의 전문 분야야! 우리는 프론트엔드가 필요한 것을 정확히 가져올 수 있게 해주는 강력한 API를 만들어. 사실 REST API를 선호하는데, 요즘 몇몇 프로젝트에서는 GraphQL로 재미를 많이 보고 있어.

### 데이터베이스
우리가 모든 것을 저장하는 곳이야! 프로젝트에 따라 MongoDB(당신의 Tonight Pass용), PostgreSQL(복잡한 관계가 필요한 프로젝트용), 때로는 캐싱을 위해 Redis를 사용해. 데이터베이스 선택은 중요하고 우리는 항상 미리 생각해.

### 비즈니스 로직
우리의 일의 핵심이야. 고객의 필요를 작동하는 코드로 변환하는 거지. 예를 들어 Tonight Pass의 추천 알고리즘을 위해, 우리는 몇 주 동안 규칙을 다듬고 쿼리를 최적화했어.

## 우리가 가장 좋아하는 백엔드 스택

### 언어
- **Node.js/TypeScript**: 우리의 최애 조합! 거의 모든 최신 프로젝트에서 사용해
- **Python**: 데이터 분석이나 머신 러닝이 필요할 때 완벽해
- **Go**: 성능이 필요한 특정 마이크로서비스에 채택했어

### 프레임워크
- **NestJS**: 큰 애플리케이션을 위한 우리의 최애 Node.js 프레임워크
- **Express**: 더 간단한 API나 프로토타입을 위해
- **Django**: 빠르게 강력한 관리자를 필요로 할 때

## 왜 이게 그렇게 중요한가

백엔드는 집의 기초와 같아. 만약 제대로 되어 있지 않으면 모든 게 무너져. onRuntime에서 우리는 특히 자랑스러워:

- Tonight Pass에서 트래픽 급증을 처리하는 우리의 아키텍처(새해 전야 파티 덕분!)
- 우리의 완벽한 보안(우드에 노크!)
- 우리의 초고속 응답 시간

고객에게는 가장 덜 보이는 부분이지만, 우리가 가장 머리 아픈 부분이야... 그리고 모든 게 순조롭게 돌아갈 때 가장 큰 만족을 줘!
