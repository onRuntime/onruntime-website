## TranslationsProvider

आपके ऐप्लिकेशन को लपेटने वाला प्रोवाइडर कंपोनेंट।

```tsx
import { TranslationsProvider } from "@onruntime/translations/react";

<TranslationsProvider
  locale="en"
  locales={{ en, fr }}
  fallbackLocale="en"
>
  {children}
</TranslationsProvider>
```

### Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `locale` | `string` | हाँ | वर्तमान सक्रिय लोकल |
| `locales` | `Record<string, object>` | हाँ | सभी लोकल अनुवादों को समाहित करने वाला ऑब्जेक्ट |
| `fallbackLocale` | `string` | नहीं | जब कुंजी गायब हो तब फॉल्बैक लोकल |
| `children` | `ReactNode` | हाँ | चाइल्ड कंपोनेंट्स |

## useTranslation

क्लाइंट कंपोनेंट्स में अनुवाद तक पहुंचने का हुक।

```tsx
const { t, locale, setLocale } = useTranslation("namespace");
```

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `namespace` | `string` | हाँ | उपयोग करने के लिए अनुवाद नामस्थान |

### Returns

| Property | Type | Description |
|----------|------|-------------|
| `t` | `(key: string, vars?: object) => string` | अनुवाद फ़ंक्शन |
| `locale` | `string` | वर्तमान लोकल |
| `setLocale` | `(locale: string) => void` | लोकल बदलने के लिए फ़ंक्शन |

### Example

```tsx
const { t, locale, setLocale } = useTranslation("common");

// बुनियादी उपयोग
t("greeting");  // "Hello"

// चर के साथ
t("greeting_name", { name: "John" });  // "Hello, John!"

// नेस्टेड कुंजी
t("nav.home");  // "Home"

// लोकल बदलें
setLocale("fr");
```

## getTranslation

सर्वर कंपोनेंट्स (Next.js) में अनुवादों तक पहुंचने के लिए असिंक्रोनस फ़ंक्शन।

```tsx
const { t } = await getTranslation("namespace");
```

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `namespace` | `string` | हाँ | उपयोग करने के लिए अनुवाद नामस्थान |

### Returns

| Property | Type | Description |
|----------|------|-------------|
| `t` | `(key: string, vars?: object) => string` | अनुवाद फ़ंक्शन |
| `locale` | `string` | वर्तमान लोकल |

### Example

```tsx
// सर्वर कंपोनेंट
export default async function Page() {
  const { t } = await getTranslation("common");

  return <h1>{t("title")}</h1>;
}
```

## Link Component

Next.js के लिए लोकल-जानकारी वाला लिंक कंपोनेंट।

```tsx
import { Link } from "@onruntime/translations/next";

<Link href="/about">About</Link>
```

### Props

Next.js के `Link` कंपोनेंट से सभी प्रोपीज़, प्लस:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `locale` | `string` | वर्तमान लोकल | लिंक लोकल को ओवरराइड करें |

### Example

```tsx
// वर्तमान लोकल का उपयोग करता है
<Link href="/about">About</Link>

// विशेष लोकल मजबूर करें
<Link href="/about" locale="fr">À propos</Link>
```

## Translation Function (t)

`t` फ़ंक्शन अनुवादित स्ट्रिंग्स प्राप्त करने के लिए उपयोग किया जाता है।

### Signature

```tsx
t(key: string, variables?: Record<string, string | number>): string
```

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `key` | `string` | अनुवाद कुंजी (डॉट नोटेशन का समर्थन करता है) |
| `variables` | `object` | इंटरपोलेट करने के लिए चर |

### Variable Interpolation

```json
{
  "greeting": "Hello, {name}!",
  "items": "You have {count} items"
}
```

```tsx
t("greeting", { name: "Alice" });  // "Hello, Alice!"
t("items", { count: 5 });          // "You have 5 items"
```

### Nested Keys

```json
{
  "user": {
    "profile": {
      "title": "Profile"
    }
  }
}
```

```tsx
t("user.profile.title");  // "Profile"
```

## Type Definitions

### TranslationNamespaces

टाइप-सेफ अनुवाद के लिए इस इंटरफेस को विस्तारित करें:

```tsx
declare module "@onruntime/translations" {
  interface TranslationNamespaces {
    common: typeof import("./locales/en/common.json");
    auth: typeof import("./locales/en/auth.json");
  }
}
```

<Callout type="info" title="TypeScript Benefits">
  सही टाइप परिभाषाओं के साथ, आपको अनुवाद कुंजी के लिए ऑटो-कम्पलीट मिलेगा और अमान्य कुंजी के लिए कंपाइल-टाइम त्रुटियाँ।
</Callout>
