## 定義

バックエンドは店舗の裏にある工場のようなもの！onRuntimeでアプリケーションを動かす見えない部分全てがそれです。例えばTonight Passを使うと、美しく直感的なインターフェースとやり取りしますが、実際にすべての重作業を行っているのはバックエンドです：あなたの認証情報を確認したり、イベントデータを保存したり、推薦を計算したりします。

## 私たちの日常業務

### サーバー
バックエンドエンジニアはサーバーの最適化に時間をかけています。Tonight Passのトラフィックが増加しているので、すべてを最適化するのに多くの時間を費やさなければなりませんでした！主にAWSとGoogle Cloudを使用してアプリケーションをホストしています。

### API
これが私たちの専門です！フロントエンドが必要なものを正確に取得できる強力なAPIを作成しています。実際、私たちはREST APIを好みますが、特定のプロジェクトではGraphQLを使ってさらに楽しい経験をしています。

### データベース
すべてを保存する場所です！プロジェクトによって、MongoDB（Tonight Pass用）、PostgreSQL（複雑なリレーションを必要とするプロジェクト用）、そしてキャッシュ用に時々Redisを使い分けています。データベースの選択は重要で、いつも事前に考えています。

### ビジネスロジック
私たちの活動の中心です。クライアントのニーズを動くコードに変えること。例えば、Tonight Passの推薦アルゴリズムのために、私たちはルールを洗練し、クエリを最適化するのに何週間も費やしました。

## 私たちのお気に入りのバックエンドスタック

### 言語
- **Node.js/TypeScript**: 私たちのお気に入りのコンボ！ほぼすべての最近のプロジェクトで使用しています
- **Python**: データ分析や機械学習を行う必要があるときに最適
- **Go**: パフォーマンスを必要とする特定のマイクロサービスのために採用しました

### フレームワーク
- **NestJS**: 大規模アプリケーション向けの私たちのお気に入りのNode.jsフレームワーク
- **Express**: よりシンプルなAPIやプロトタイプ用
- **Django**: 迅速に堅牢な管理画面が必要なとき

## それがなぜ重要なのか

バックエンドは家の基礎のようなものです。しっかりと作られていないと、全てが崩れます。onRuntimeでは以下の点に特に誇りを持っています：

- Tonight Passのトラフィックの急増を処理するアーキテクチャ（新年のパーティーのおかげ！）
- 完璧なセキュリティ（木にノック）
- 超高速の応答時間

クライアントにとっては最も目に見えない部分ですが、最も頭痛のタネになる部分でもあります...そしてすべてがスムーズに動くときの満足感も最も高いです！
