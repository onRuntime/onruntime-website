## Czym jest @onruntime/translations?

`@onruntime/translations` to lekka, typowo bezpieczna biblioteka międzynarodowa (i18n) zaprojektowana dla aplikacji React i Next.js. Oferuje prosty API do zarządzania tłumaczeniami z pełnym wsparciem dla TypeScript.

<Callout type="info" title="Kluczowe cechy">
  - Typowo bezpieczne tłumaczenia z TypeScript
  - Wsparcie dla React i Next.js (App Router)
  - Organizacja oparta na przestrzeniach nazw
  - Interpolacja zmiennych
  - Lekka i wydajna
</Callout>

## Czemu warto z tego skorzystać?

### Typowa bezpieczeństwo

Uzyskaj błędy w czasie kompilacji, gdy używasz nieprawidłowych kluczy tłumaczeń:

```typescript
const { t } = useTranslation("common");

t("greeting");        // ✅ Prawidłowy klucz
t("invalid_key");     // ❌ Błąd TypeScript
```

### Proste API

Tylko dwa haki do wszystkich twoich potrzeb tłumaczeniowych:

```typescript
// Komponenty Klienta
const { t } = useTranslation("namespace");

// Komponenty Serwera (Next.js)
const { t } = await getTranslation("namespace");
```

### Zorganizowane tłumaczenia

Utrzymuj swoje tłumaczenia zorganizowane w przestrzeniach nazw:

<Files>
  <Folder name="locales" defaultOpen>
    <Folder name="en" defaultOpen>
      <File name="common.json" />
      <File name="auth.json" />
      <File name="dashboard.json" />
    </Folder>
    <Folder name="fr">
      <File name="common.json" />
      <File name="auth.json" />
      <File name="dashboard.json" />
    </Folder>
  </Folder>
</Files>

## Szybki przykład

```tsx
// locales/en/common.json
{
  "greeting": "Cześć, {name}!"
}

// Komponent
import { useTranslation } from "@onruntime/translations/react";

function Welcome({ name }) {
  const { t } = useTranslation("common");

  return <h1>{t("greeting", { name })}</h1>;
}
```
