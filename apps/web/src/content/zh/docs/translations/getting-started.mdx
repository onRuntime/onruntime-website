## 什么是 @onruntime/translations?

`@onruntime/translations` 是一个轻量级的，类型安全的国际化（i18n）库，专为 React 和 Next.js 应用设计。它提供了一个简单的 API，用于管理翻译，并支持完整的 TypeScript。

<Callout type="info" title="主要特点">
  - 使用 TypeScript 的类型安全翻译
  - 支持 React 和 Next.js（App Router）
  - 基于命名空间的组织
  - 变量插值
  - 轻量且高效
</Callout>

## 为什么使用它？

### 类型安全

使用无效的翻译键时会获得编译时错误：

```typescript
const { t } = useTranslation("common");

t("greeting");        // ✅ 有效的键
t("invalid_key");     // ❌ TypeScript 错误
```

### 简单的 API

仅需两个钩子即可满足所有翻译需求：

```typescript
// 客户端组件
const { t } = useTranslation("namespace");

// 服务器组件（Next.js）
const { t } = await getTranslation("namespace");
```

### 有序的翻译

使用命名空间保持翻译的有序性：

<Files>
  <Folder name="locales" defaultOpen>
    <Folder name="en" defaultOpen>
      <File name="common.json" />
      <File name="auth.json" />
      <File name="dashboard.json" />
    </Folder>
    <Folder name="fr">
      <File name="common.json" />
      <File name="auth.json" />
      <File name="dashboard.json" />
    </Folder>
  </Folder>
</Files>

## 快速示例

```tsx
// locales/en/common.json
{
  "greeting": "你好，{name}!"
}

// 组件
import { useTranslation } from "@onruntime/translations/react";

function Welcome({ name }) {
  const { t } = useTranslation("common");

  return <h1>{t("greeting", { name })}</h1>;
}
```
