## परिभाषा

एक API (एप्लिकेशन प्रोग्रामिंग इंटरफ़ेस) मूल रूप से वह भाषा है जिसका हमारे एप्लिकेशन एक-दूसरे से संवाद करने के लिए उपयोग करते हैं। onRuntime में, हम इन्हें लगातार अपने परियोजनाओं के विभिन्न हिस्सों को जोड़ने के लिए इस्तेमाल करते हैं। एक API को एक रेस्तरां में वेटर की तरह सोचें। आप एक ऑर्डर देते हैं (अनुरोध), वो आपको वो लाते हैं जो आपने मांगा (प्रतिक्रिया), और आपको नहीं पता होता कि किचन में क्या हो रहा है।

## हम APIs को क्यों पसंद करते हैं

- **ये सब कुछ एक साथ काम कराते हैं**: विभिन्न सिस्टम बिना किसी परेशानी के डेटा का आदान-प्रदान कर सकते हैं
- **ये हमें मॉड्यूलर बनाते हैं**: हम स्वतंत्र घटक विकसित कर सकते हैं और उन्हें आसानी से जोड़ सकते हैं
- **ये हमारे डेटा की सुरक्षा करते हैं**: हम नियंत्रण करते हैं कि कौन क्या एक्सेस करता है
- **ये हमारा समय बचाते हैं**: हम एक हिस्से को अपडेट कर सकते हैं बिना सब कुछ फिर से बनाए

## APIs जो हम रोज़ इस्तेमाल करते हैं

### RESTful APIs
हमारी दैनिक रोटी! हम इन्हें अपनी ज्यादातर वेब परियोजनाओं के लिए इस्तेमाल करते हैं। ये सरल, स्टेटलेस (एक अनुरोध से दूसरे में कोई याददाश्त नहीं) हैं, और क्लासिक HTTP तरीकों का उपयोग करते हैं। जैसा कि Tonight Pass पर के लिए सही है।

### GraphQL APIs
हम इनसे कुछ जटिल परियोजनाओं के लिए पंसद करते हैं! 15 विभिन्न एंडपॉइंट बनाने के बजाय, हम क्लाइन्ट को ठीक वही डेटा मांगने देते हैं जिसकी उसे जरूरत है। Kitchn पर, इसने हमें बहुत ज्यादा बैंडविड्थ बचाने में मदद की।

### SOAP APIs
हम झूठ नहीं बोलेंगे, अब हम इनका ज्यादा उपयोग नहीं करते... ये अपने XML और अल्ट्रा-सख्त विनिर्देशों के साथ थोड़ा पुराने स्कूल हैं। लेकिन हम still कभी-कभी इनसे एंटरप्राइज परियोजनाओं पर मिलते हैं।

## हम इन्हें रोज़ कैसे इस्तेमाल करते हैं

onRuntime में, APIs हमारे तकनीकी स्टैक के दिल में हैं। ये हमें अनुमति देती हैं:

- अलग-अलग एप्लिकेशन बनाना (अलग फ्रंटेंड/बैकेंड)
- तीसरे पक्ष की सेवाओं को आसानी से एकीकृत करना (भुगतान, मैपिंग, आदि)
- कुछ परियोजनाओं पर हमारी माइक्रोसर्विस आर्किटेक्चर विकसित करना

जब हम एक API डिजाइन करते हैं, तो हम हमेशा इसे Swagger या Postman के साथ दस्तावेजीकरण करके शुरू करते हैं। इससे हमें चीज़ों के बारे में विचार करने में मदद मिलती है और यह टीम के लिए एक ठोस संदर्भ बन जाता है। मुझ पर विश्वास करो, यह कई बेकार बैठकों को बचाता है!
